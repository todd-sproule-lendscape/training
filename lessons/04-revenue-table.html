Revenue Table & Configuration

What is the Revenue Table?

Think of the revenue table as a spreadsheet that shows what happens financially in each period of a lease. Every row is one accounting period, and the columns show things like how much income was earned, how much the asset depreciated, and what its current value is.

Standard Revenue Table Columns

These columns appear for every lease, whether operating or finance:

Column Description
start_dateStart date of the accounting period
end_dateEnd date of the accounting period
daysNumber of days in the period
rental_incomeStraight-line rental income for the period
depreciationDepreciation charge for the period
asset_net_book_valueNet book value of the asset at period end

Extra Columns for Operating Leases

Operating leases need a few more columns to track things that don't apply to finance leases:

Column Description Purpose
asset_cash_flow_event Money spent on or received for the asset itself Records when the lessor buys the asset at the start and sells it at the end
rental_income_cash_flow_event Actual rent money received The cash the tenant actually pays each period — this can differ from the straight-line income figure
charge_cash_flow_event Cash for fees and charges (e.g., IDCs) Records payments for set-up costs like broker fees or legal expenses
rental_prepayment_or_accrual Running difference between cash and income If you've received more cash than income recorded, that's a prepayment (positive). If less, it's an accrual (negative). Over the full lease, this balances to zero.

Example Revenue Table

For a 3-period operating lease: Asset cost £60,000, residual £12,000, rental £1,800/month, IDC £900:

Period Start End Days Rental Income Depreciation IDC Expense NBV Prepay/Accrual
101 Jan31 Jan31 £1,800£1,600£300 £58,400£0
201 Feb28 Feb28 £1,800£1,600£300 £56,800£0
301 Mar31 Mar31 £1,800£1,600£300 £55,200£0

Depreciation: (£60,000 − £12,000) ÷ 30 months = £1,600/month. IDC: £900 ÷ 3 = £300/month.

RIAD Output Records: ALIR & ALCR

When RIAD runs, it produces two types of records that feed into the revenue table. Think of them as two “building blocks” that get combined to form each row:

ALIR — Asset Level Interest Revenue

Covers the core lease economics:

  • Rental income (straight-line)
  • Unearned rental income (remaining balance)
  • Book depreciation
  • Unrealised depreciation (remaining balance)
  • Asset net book value
  • Cash flow events (asset & rental)
ALCR — Asset Level Charge Revenue

Covers fee and charge components (e.g., IDCs):

  • Charge amount (straight-line amortisation)
  • Unearned charge (remaining balance)
  • Charge cash flow events
  • One ALCR record per charge type per period

Mapping RIAD Output to Revenue Table

Revenue Table Column Source Description
rental_income ALIR Straight-line rental income for the period (daily rate × days)
depreciation ALIR Book depreciation for the period (daily rate × days)
asset_net_book_value ALIR Asset cost minus cumulative depreciation
asset_cash_flow_event ALIR Asset purchase/disposal cash flows
rental_income_cash_flow_event ALIR Cash received from lessee for rentals
charge_cash_flow_event ALCR Cash flows for charges (IDCs, fees)
rental_prepayment_or_accrual Derived Cumulative difference between cash received and income recognised
How they fit together: Each period gets one ALIR (for the main lease numbers) and one ALCR per charge (e.g., one for IDCs). The revenue table row is simply these records combined side by side.

System Configuration

accounting-agreement-types

This setting tells the system which accounting method to use for a lease. For operating leases, it's always:

RENTAL_INCOME_AND_DEPRECIATION

This is the method configured for operating leases. It instructs the system to:

  • Recognise rental income on a straight-line basis
  • Calculate depreciation on the underlying asset
  • Amortise IDCs over the lease term

charge-accounting-types

Each charge (rental income, depreciation, IDC) has its own type that controls how it's calculated. The three types you'll see for operating leases:

Charge Accounting Type Description
STRAIGHT_LINE_RENTAL_INCOME Recognises rental income evenly across the lease term. Generates journal entries for rental income and any prepayment/accrual balance.
STRAIGHT_LINE_DEPRECIATION Calculates straight-line depreciation from asset cost to residual value over the useful life. Generates depreciation expense journal entries.
STRAIGHT_LINE_CHARGE Used for IDC amortisation. Spreads the charge evenly across the lease term. Generates amortisation expense journal entries.

How It All Connects

  1. Lease Agreement is linked to an accounting-agreement-type (e.g., RENTAL_INCOME_AND_DEPRECIATION)
  2. The accounting-agreement-type defines the charges to generate (rental income, depreciation, IDC)
  3. Each charge has a charge-accounting-type (e.g., STRAIGHT_LINE_RENTAL_INCOME)
  4. The charge-accounting-type determines the calculation method and the journal entry template
  5. The revenue table is populated based on these calculations

Key Takeaways

  • The revenue table is a period-by-period schedule of income, depreciation, and net book value
  • Operating leases add columns for asset cash flows, rental cash flows, charge cash flows, and prepayment/accrual tracking
  • RIAD produces ALIR records (core lease: rental income, depreciation, NBV) and ALCR records (charges: IDC amortisation, fees)
  • The RENTAL_INCOME_AND_DEPRECIATION method drives the operating lease accounting
  • Three charge-accounting-types handle the different calculations: rental income, depreciation, and IDC amortisation
